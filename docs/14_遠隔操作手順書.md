**対象:** Dさんの Windows PC（CrossFactor 稼働環境）  
**担当:** 中野晃志（開発側） / Dさん（クライアント側）  
**作成日:** 2026-06-29  
**ステータス:** Draft（接続テスト実施後に `Decided` へ更新）

## 1. 概要

本手順書は、クライアント PC に **AnyDesk がインストール済み** の状態から、開発側が遠隔で作業するための接続方法をまとめたものです。

| 接続の種類 | 用途 | いつ使うか |
|-----------|------|-----------|
| **AnyDesk** | 画面共有・初期セットアップ・GUI 操作 | 初回設定、SSH 未整備時、画面確認が必要なとき |
| **SSH（Tailscale 経由）** | コマンド実行・バッチ起動・ファイル転送 | 日常運用（推奨） |
| **CrossFactor SQLite** | 競馬データの読み取り元 | バッチ抽出・推論の入力 |
| **Supabase** | Web 表示用クラウド DB | 予測結果の反映・フロント確認 |
| **GCP VM PostgreSQL** | AI 用バックアップ DB（任意） | フェイルオーバー・クラウド側 AI 処理 |

### 1.1 接続の全体像

```
中野晃志の PC
    │
    ├─[AnyDesk]──────────────► Dさんの Windows PC
    │                              ├─ CrossFactor / SQLite
    │                              ├─ keiba-ai バッチ
    │                              └─ OpenSSH Server
    │
    ├─[Tailscale VPN + SSH]──► Dさんの Windows PC（日常運用）
    │
    ├─[HTTPS]────────────────► Supabase（クラウド DB）
    │
    └─[gcloud ssh]───────────► GCP VM keiba-ai-vm（バックアップ AI DB）
```

**運用の基本方針**

1. **初回・設定作業** → AnyDesk で画面を見ながら SSH / Tailscale を整備する  
2. **日常のバッチ実行・ログ確認** → SSH でコマンド操作する（軽量・安全）  
3. **GUI が必要なとき**（CrossFactor 設定確認など）→ AnyDesk または Tailscale 経由 RDP

## 2. 事前準備（両者共通）

### 2.1 共有しておく情報

接続テストの前に、以下をチャット等で共有してください。

| 項目 | 共有者 | 例 |
|------|--------|-----|
| AnyDesk アドレス（9桁） | Dさん → 中野 | `123 456 789` |
| AnyDesk 無人アクセス用パスワード（設定した場合） | Dさん → 中野 | （各自で安全な経路で共有） |
| Windows ログインユーザー名 | Dさん → 中野 | `DShibata` など |
| CrossFactor SQLite のパス | Dさん → 中野 | `C:\CrossFactor\data\xxx.db` |
| 作業可能な時間帯 | 双方 | 例: 平日 10:00〜18:00 |

### 2.2 セキュリティ上の注意

- AnyDesk のパスワードは **推測されにくいもの** にし、定期的に見直す  
- 接続が不要な時間帯は AnyDesk を終了するか、無人アクセスをオフにする  
- `.env` や API キーは **チャットに貼らない**（画面共有時もマスクする）  
- SSH は公開鍵認証を設定し、パスワードログインは最小限にする

## 3. AnyDesk 接続手順

> **前提:** Dさんの PC に AnyDesk はインストール済み

### 3.1 Dさん側（クライアント）— 接続を受け入れる

1. AnyDesk を起動する  
2. 画面上部の **「このデバイス」** に表示される **9桁のアドレス** を中野に伝える  
3. 接続リクエストが来たら、画面に表示される **「承認」** をクリックする  
4. （推奨）無人アクセスを使う場合:
   - `設定` → `セキュリティ` → `無人アクセスを有効にする`
   - 強力なパスワードを設定し、中野に安全な経路で共有する

### 3.2 中野側（開発）— 接続する

1. [AnyDesk 公式サイト](https://anydesk.com/ja) からクライアントをインストール（未導入の場合）  
2. 「リモートデスクアドレス」欄に Dさんのアドレスを入力  
3. `接続` をクリック  
4. パスワード入力または Dさんの承認を待つ  
5. 接続後、以下を確認:
   - デスクトップが正常に表示される  
   - マウス・キーボード操作ができる  
   - 切断後に再接続できる

### 3.3 AnyDesk 経由で行う初期作業

SSH や Tailscale のセットアップは、**AnyDesk で画面を共有した状態** で行うとスムーズです。

| 作業 | 操作場所 |
|------|---------|
| Tailscale のインストール | Dさんの PC（AnyDesk 画面内） |
| OpenSSH Server の有効化 | Dさんの PC（管理者 PowerShell） |
| `keiba-ai` リポジトリの配置確認 | Dさんの PC |
| `config.yaml` の SQLite パス確認 | Dさんの PC |
| `.env` の配置（Supabase キー等） | Dさんの PC |

## 4. SSH 接続手順（Tailscale + OpenSSH）

日常運用では **Tailscale VPN 上の SSH** を標準とします。詳細は `SSH接続手順書.md` も参照してください。

### 4.1 Tailscale のインストール（両 PC）

1. https://tailscale.com/download から Windows 用インストーラを取得  
2. 両 PC にインストールし、**同じ Tailscale アカウント** でサインイン  
3. 接続確認:

```powershell
tailscale ip -4
# → 100.x.x.x 形式の IP が表示されれば OK
```

### 4.2 OpenSSH Server の有効化（Dさんの PC）

AnyDesk 接続中に、**管理者権限の PowerShell** で実行します。

```powershell
# OpenSSH Server をインストール
Add-WindowsCapability -Online -Name OpenSSH.Server~~~~0.0.1.0

# サービスを起動・自動起動に設定
Start-Service sshd
Set-Service -Name sshd -StartupType 'Automatic'

# ファイアウォールルール確認
Get-NetFirewallRule -Name "OpenSSH*"
```

### 4.3 SSH 接続テスト（中野の PC から）

```powershell
# Dさんの Tailscale IP と Windows ユーザー名を指定
ssh DさんのWindowsユーザー名@100.x.x.x

# 接続後の確認
python --version
cd C:\keiba-ai
dir
```

### 4.4 公開鍵認証の設定（推奨）

```powershell
# 中野の PC で鍵ペア生成（既にある場合はスキップ）
ssh-keygen -t ed25519 -C "keiba-ai-ops"

# 公開鍵を Dさんの PC に登録
type $env:USERPROFILE\.ssh\id_ed25519.pub | ssh DさんのWindowsユーザー名@100.x.x.x "mkdir -Force $env:USERPROFILE\.ssh; Add-Content -Force -Path $env:USERPROFILE\.ssh\authorized_keys -Value '$(Get-Content -Raw $env:USERPROFILE\.ssh\id_ed25519.pub)'"
```

### 4.5 よく使う SSH コマンド

```powershell
# バッチ dry-run
ssh DさんのWindowsユーザー名@100.x.x.x "C:\keiba-ai\.venv\Scripts\python.exe C:\keiba-ai\batch\run_pipeline.py --mode daily_evening --dry-run"

# 本番バッチ実行
ssh DさんのWindowsユーザー名@100.x.x.x "C:\keiba-ai\.venv\Scripts\python.exe C:\keiba-ai\batch\run_pipeline.py --mode daily_evening"

# ログを手元に取得
scp DさんのWindowsユーザー名@100.x.x.x:C:/keiba-ai/logs/batch_*.log ./logs/
```

## 5. データベース接続手順

### 5.1 CrossFactor SQLite（Dさんの PC 上）

競馬データの **読み取り元** です。バッチは読み取り専用で接続します。

#### パスの確認

1. AnyDesk または SSH で Dさんの PC に接続  
2. CrossFactor のデータ保存場所を確認（例: `C:\CrossFactor\data\`）  
3. `config.yaml` の `sqlite.path` を実パスに合わせる:

```yaml
sqlite:
  path: "C:/CrossFactor/data/実際のファイル名.db"
```

#### 接続テスト

```powershell
# SSH または AnyDesk 経由で Dさんの PC 上で実行
cd C:\keiba-ai
.\.venv\Scripts\python.exe -c "
from extract.sqlite_client import open_db
import yaml
with open('config.yaml') as f:
    cfg = yaml.safe_load(f)
path = cfg['sqlite']['path']
with open_db(path) as conn:
    cur = conn.execute('SELECT name FROM sqlite_master WHERE type=\"table\" LIMIT 5')
    print([r[0] for r in cur.fetchall()])
print('SQLite 接続 OK')
"
```

**注意事項**

- 接続テスト中は可能な限り **CrossFactor を閉じる**（ファイルロック回避）  
- `.db-wal` / `.db-shm` が隣にある場合は、CrossFactor 稼働中でも WAL モードで読めることが多いが、エラー時は再起動を試す  
- 初回データ受領手順は `18_クライアント向けDBアップロード手順.md` を参照

---

### 5.2 Supabase（クラウド DB）

Web フロントとバッチの **同期先** です。

#### 環境変数の設定（Dさんの PC）

`C:\keiba-ai\.env` に以下を設定します（値は Supabase ダッシュボードから取得）。

```bash
SUPABASE_URL=https://xxxx.supabase.co
SUPABASE_SERVICE_ROLE_KEY=（service_role キー。フロントには公開しない）
```

#### 接続テスト

```powershell
cd C:\keiba-ai
.\.venv\Scripts\python.exe -c "
from load.supabase_client import create_client
client = create_client()
result = client.table('entries').select('id').limit(1).execute()
print('Supabase 接続 OK:', len(result.data), '件')
"
```

#### ブラウザでの確認

1. https://supabase.com/dashboard にログイン  
2. プロジェクト `crossfactor-ai-keiba`（または該当プロジェクト）を開く  
3. `Table Editor` で `entries` / `races` 等のレコードを確認  
4. バッチ実行後に `updated_at` や予測カラムが更新されていることを確認

---

### 5.3 GCP VM PostgreSQL（AI 用バックアップ DB・任意）

フェイルオーバー用の AI 処理 DB です。通常の日次運用では Dさん PC 上のバッチ → Supabase が主経路です。

#### SSH で VM に接続

```bash
gcloud compute ssh keiba-ai-vm --zone=us-central1-a --project=crossfactor-ai-keiba
```

#### PostgreSQL 接続確認（VM 上）

```bash
export PGPASSWORD="（.env の AI_DB_PASSWORD）"
psql -h localhost -U ai_runner -d keiba_ai -c "\dt"
```

期待されるテーブル: `raw_races` / `raw_entries` / `features` / `predictions_staging` / `job_runs`

詳細は `GCP_VM_構築手順書.md` を参照してください。

## 7. トラブルシューティング

### AnyDesk

| 症状 | 想定原因 | 対処 |
|------|---------|------|
| 接続できない | AnyDesk 未起動 | Dさんに AnyDesk を起動してもらう |
| 承認が来ない | 通知オフ・別画面 | Dさんに AnyDesk ウィンドウを確認してもらう |
| 画面が重い | 回線速度 | 解像度を下げる（表示 → 画質調整） |

### SSH / Tailscale

| 症状 | 想定原因 | 対処 |
|------|---------|------|
| SSH タイムアウト | OpenSSH 停止 | Dさん PC で `Start-Service sshd` |
| Tailscale に到達できない | VPN 未接続 | 両 PC で Tailscale アプリを起動 |
| 認証失敗 | ユーザー名誤り | Windows アカウント名を再確認 |
| `python` が見つからない | PATH 未設定 | `C:\keiba-ai\.venv\Scripts\python.exe` をフルパス指定 |

### データベース

| 症状 | 想定原因 | 対処 |
|------|---------|------|
| SQLite ロックエラー | CrossFactor 起動中 | CrossFactor を一度閉じて再試行 |
| SQLite ファイル未検出 | パス誤り | `config.yaml` の `sqlite.path` を修正 |
| Supabase 401/403 | キー誤り・期限切れ | `.env` の `SUPABASE_SERVICE_ROLE_KEY` を再設定 |
| Supabase にデータが出ない | バッチ失敗 | `C:\keiba-ai\logs\` のログを確認 |

## 8. 緊急時・エスカレーション

| 状況 | 対応 |
|------|------|
| SSH が使えずバッチが止まった | まず AnyDesk で接続し、手動でバッチ実行またはログ確認 |
| CrossFactor / SQLite にアクセス不能 | Dさんに CrossFactor 再起動を依頼。改善しなければ `18_クライアント向けDBアップロード手順.md` でデータ再送を依頼 |
| Supabase 障害 | GCP VM 側のバックアップ DB で一時運用（`GCP_VM_構築手順書.md` §フェイルオーバー参照） |
| 接続手段がすべて使えない | 電話 / チャットで状況共有。次回接続可能時間を調整 |

## 1. モデル概要

### 1.1 モデル構成

| モデル名 | ファイル名 | 予測ターゲット | アルゴリズム |
|---------|-----------|-------------|------------|
| 勝率モデル | `lgbm_win_v1.pkl` | `is_win`（1着かどうか） | LightGBM Binary Classification |
| 連対モデル | `lgbm_top2_v1.pkl` | `is_top2`（2着以内かどうか） | LightGBM Binary Classification |
| 複勝モデル | `lgbm_top3_v1.pkl` | `is_top3`（3着以内かどうか） | LightGBM Binary Classification |

### 1.2 予測の流れ

```
CrossFactor SQLite
        │
        ▼
  extract/ : SQLクエリでデータ抽出
        │
        ▼
  transform/ : feature_mapper（日本語→英語列名変換）
             + normalizer（欠損補完・ラベルエンコード）
             + odds_selector（最新オッズ選択）
        │
        ▼
  features.parquet（特徴量ファイル）
        │
        ▼
  inference/run_inference.py
    ├── lgbm_win_v1.pkl   → raw_win（生確率）
    ├── lgbm_top2_v1.pkl  → raw_top2（生確率）
    └── lgbm_top3_v1.pkl  → raw_top3（生確率）
        │
        ▼
  レース内正規化（合計→100%に揃える）
        │
        ▼
  期待値算出・star_rating付与
        │
        ▼
  Supabase entries テーブルへ upsert
```

### 1.3 ハイパーパラメータ

```python
LGB_PARAMS = {
    "objective":         "binary",
    "metric":            "auc",
    "boosting_type":     "gbdt",
    "num_leaves":        63,
    "learning_rate":     0.05,
    "feature_fraction":  0.8,
    "bagging_fraction":  0.8,
    "bagging_freq":      5,
    "min_child_samples": 20,
    "n_estimators":      1000,
    "random_state":      42,
    "verbose":           -1,
}
```

Early Stopping: `callbacks=[lgb.early_stopping(50)]`（バリデーションAUCで50ラウンド改善なしで停止）

```
df_entries（推論時の入力DataFrame）:
  必須列: entry_key, race_key, horse_number + 上記 FEATURE_COLS
  行単位: 1行 = 1頭の出走馬
  欠損: 存在しない特徴量列は 0 で自動補完（run_inference.py 内）
```

### 2.3 正解ラベル（学習時のみ）

| ラベル名 | 型 | 計算方法 | 取得元 |

| `is_top2` | int（0/1） | `finish_position <= 2` → 1、それ以外 → 0 | JV馬毎 / `確定着順` |

## 3. 出力データ仕様

### 3.1 推論出力（`inference/run_inference.py` の `run()` 戻り値）

`run()` は `Optional[pd.DataFrame]` を返す。失敗時は `None`。

#### 3.1.1 基本予測値（レース内正規化済み）

| カラム名 | 型 | 単位 | 説明 | 算出方法 |
|---------|---|-----|-----|---------|
| `win_rate` | float | % | 予想勝率 | `raw_win / sum(raw_win) × 100`（レース内正規化） |
| `place_rate` | float | % | 予想連対率（2着以内） | `raw_top2 / sum(raw_top2) × 100`（レース内正規化） |
| `show_rate` | float | % | 予想複勝率（3着以内） | `raw_top3 / sum(raw_top3) × 100`（レース内正規化） |

- 値域: `[0.0, 100.0]`（`.clip(0, 100).round(1)`）
- 同一レース内の合計値は理論上 100.0% に近づく（正規化後）

#### 3.1.2 期待値

| カラム名 | 型 | 単位 | 説明 | 算出式 |
|---------|---|-----|-----|-------|
| `expected_value_win` | float | 倍 | 単勝期待値 | `(win_rate / 100) × win_odds` |
| `expected_value_place` | float | 倍 | 複勝期待値 | `(show_rate / 100) × place_odds_low` |

- 値域: `[0.0, ∞)`（`.round(3)`）
- 期待値 ≥ 1.0 が「買い」の目安（要件定義書 §12 用語集より）

#### 3.1.3 キー列（出力に含まれる識別子）

| カラム名 | 型 | 説明 |
|---------|---|-----|
| `entry_key` | str | 出走馬の一意キー（18桁：`race_key` + 馬番2桁） |
| `race_key` | str | レースの一意キー（16桁） |
| `horse_number` | int | 馬番 |
| `win_odds` | float | 単勝オッズ（入力値をそのまま引き継ぎ） |
| `place_odds_low` | float | 複勝オッズ下限（入力値をそのまま引き継ぎ） |
| `popularity_rank` | int | 人気順（入力値をそのまま引き継ぎ） |

### 3.2 star_rating（`assign_star_ratings()` 追加出力）

| 値 | 記号 | 意味 | 条件 |
|----|------|-----|------|
| `★` | 本命 | 期待値最大の馬 | `expected_value_win` が最大（レース内1頭） |
| `▲` | 対抗 | 2番目に期待値が高い馬 | ★以外で `expected_value_win` が2位（レース内1頭） |
| `⚠` | 危険人気馬 | 人気上位で期待値が低い馬 | `popularity_rank <= 3` かつ `expected_value_win < 1.02` |
| `◆` | 穴馬 | 人気薄で期待値が高い馬 | `popularity_rank >= 5` かつ `expected_value_win >= 1.05` |
| `None` | — | 上記以外の馬 | — |

しきい値は `config.yaml` の `inference` セクションで設定変更可能:

```yaml
inference:
  star_ev_threshold: 1.05        # ★ / ◆ の EV 基準
  triangle_ev_threshold: 1.02    # ▲ / ⚠ の EV 基準
  longshot_popularity_rank: 5    # ◆ の最低人気順位
  risky_favorite_rank: 3         # ⚠ の最大人気順位
```

### 3.3 Supabase への書き込み列

`load/supabase_publisher.py` が `entries` テーブルの以下の列のみを upsert する:

| 列名 | 説明 |
|-----|-----|
| `win_rate` | 予想勝率（%） |
| `place_rate` | 予想連対率（%） |
| `show_rate` | 予想複勝率（%） |
| `expected_value_win` | 単勝期待値 |
| `expected_value_place` | 複勝期待値 |
| `star_rating` | ★ / ▲ / ⚠ / ◆ / null |

## 4. 前処理パイプライン

### 4.1 列名変換（`transform/feature_mapper.py`）

CrossFactor SQLite の日本語カラム名を英語の特徴量名に変換する。

| 処理関数 | 入力 | 出力 |
|---------|-----|-----|
| `map_race_columns(row)` | JVレース詳細EX の行（dict） | レース特徴量 dict |
| `map_entry_columns(row)` | JV馬毎レース情報EX の行（dict） | 馬特徴量 dict |
| `map_odds_columns(row)` | JVオッズ_単複枠 の行（dict） | オッズ dict（÷10 変換含む） |

**オッズ変換の注意点:** DB には「倍 × 10」の整数で格納されているため、`map_odds_columns` で ÷10 を行う。

```
DB値 "150" → 15.0倍 / DB値 "35" → 3.5倍
```

### 4.2 欠損値補完（`transform/normalizer.py`）

| 対象列 | 欠損時の処理 | 追加フラグ |
|-------|-----------|---------|
| 調教タイム系（`slope_*`, `wood_*`） | `-999` で補完 | `{列名}_missing = 1` |
| 過去成績系（`avg_finish_*`, `avg_popularity_*`） | 列の平均値で補完 | `is_debut = 1` |
| 適性スコア・回収率系（`place_rate_*`, `*_recovery_rate`） | `0` で補完 | なし |
| 推論時に存在しない特徴量列 | `0` で補完（`run_inference.py` 内） | なし |

### 4.3 カテゴリ変数エンコード（`transform/normalizer.py`）

| 変数 | エンコード方法 | 値 |
|-----|-------------|---|
| `track_type` | Label Encoding | 芝=0 / ダート=1 / 障害=2 |
| `track_condition` | 順序エンコード | 良=0 / 稍重=1 / 重=2 / 不良=3 |
| `predicted_running_style` | Label Encoding | 逃=1 / 先=2 / 差=3 / 追=4 / 不明=0 |
| `horse_sex` | Label Encoding | 牡=0 / 牝=1 / 騸=2 / 不明=0 |

### 4.4 キー生成（`transform/key_builder.py`）

| キー | 形式 | 桁数 | 例 |
|-----|------|:---:|---|
| `race_key` | `YYYY` + `MMDD` + `競馬場コード(2桁)` + `開催回(2桁)` + `開催日目(2桁)` + `レース番号(2桁)` | 16桁 | `2026033005010101` |
| `entry_key` | `race_key` + `馬番(2桁ゼロパディング)` | 18桁 | `202603300501010103` |

### 4.5 最新オッズ選択（`transform/odds_selector.py`）

同一馬の複数タイムスタンプのオッズから、`odds_published_at` が最新のものを選択する。

```
事後オッズ使用禁止: 締め切り直前（最終発表分）のみ使用すること
```

## 5. テスト方針

### 5.1 テストレベルの定義

| レベル | 目的 | 実施タイミング |
|-------|-----|------------|
| **ユニットテスト** | 個々のモジュール関数が仕様どおりに動作することを確認 | コード変更のたびに自動実行（pytest） |
| **統合テスト** | パイプライン全体（SQLite → 推論 → 出力）が連携して正しく動くことを確認 | バッチ修正・モデル更新時 |
| **バックテスト** | 過去データで回収率105%以上を確認 | モデル学習後・MVPリリース前 |

### 5.2 テストコマンド

```bash
# 全ユニットテスト実行
pytest tests/ -v

# カバレッジ計測
pytest tests/ --cov=transform --cov=inference --cov-report=term-missing

# 特定モジュールのみ
pytest tests/test_normalizer.py -v
pytest tests/test_feature_mapper.py -v
```

## 8. バックテスト仕様

### 8.1 データ分割

```
└── テストデータ  : 2024年1月1日〜（TEST_FROM = "20240101"、一切学習に使わない）
```

### 8.2 モデル評価指標

学習スクリプト（`training/train_models.py`）で自動出力:

| 指標 | 計算タイミング | 目安 |

## 9. データリーク防止チェックリスト

学習・バックテストを実施する前に以下をすべて確認すること。

### ✅ 実装済みの防止策

- [x] **時系列分割:** `race_key` の先頭8桁（YYYYMMDD）で学習/バリデーション/テストを分割（ランダム分割禁止）
- [x] **オッズ取得タイミング:** `odds_selector.py` が締め切り直前（最終タイムスタンプ）のオッズのみを選択
- [x] **確定払戻金の排除:** 正解ラベルは `finish_position`（着順）のみを使用。払戻金は特徴量に含めない

### ⚠️ 運用時に注意が必要なポイント

| リスク | 具体的なNG操作 | 確認方法 |
|-------|------------|---------|
| 事後オッズ混入 | 締め切り後に更新されたオッズを使用する | `odds_published_at` がレース発走時刻より前であることを確認 |
| ターゲットリーク | `finish_position` や払戻金を特徴量として使用する | `FEATURE_COLS` に `finish_position`, `payout` が含まれていないことを確認 |
| 未来データの混入 | バリデーション/テストデータを使って前処理パラメータ（平均値等）を計算する | `fill_missing` の平均値は学習データのみで計算すること |
| 当日馬体重のリーク | 全頭の馬体重が発表される前に当日データを使用する | `horse_weight` は発表済み分のみ取得すること |

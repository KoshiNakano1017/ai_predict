> **位置づけ**

## 2. 勝率・期待値の計算式

```
            0.75
暗黙勝率 = ──────────── × 100        ← オッズ反転（控除率25%補正）
            win_odds

             ┌ 1〜2番人気  : × 0.92   （過剰人気を割引）
             │ 3〜4番人気  : × 0.97
AI補正勝率 = 暗黙勝率 × ┤ 5〜6番人気  : × 1.05   （中穴を加点）
             │ 7〜10番人気 : × 1.20
             └ 11番人気〜  : × 1.40   （大穴を加点）

期待値(EV)   = (AI補正勝率 / 100) × win_odds
複勝率       = 勝率 × 1.7   （上限 80%）
3着内率      = 勝率 × 2.3   （上限 90%）
期待値(複勝) = 期待値(EV) × 0.95
```

**前提となる考え方**: 「人気馬は実力以上に買われ（過剰人気）、人気薄は過小評価される」という仮説に基づき、人気下位ほど勝率を上振れ補正する。

## 3. 星マークの割り当てルール（レースごと）

| マーク | 意味 | 条件 | 最大数 |
|:---:|---|---|:---:|
| ★ | 本命 | EV が最高の1頭 | 1頭 |
| ▲ | 対抗 | ★以外で EV ≥ 1.0 の上位 | 2頭 |
| ⚠ | 危険な人気馬 | 1〜3番人気 かつ EV < 0.9 | 1頭 |
| ◆ | 穴馬 | 7番人気以下 かつ EV ≥ 1.05 | 2頭 |

割り当て順序: **★ → ▲ → ⚠ → ◆**（先に確定したマークの馬は後続の対象外）

## 5. 既知の問題点

### 5.1 穴馬（◆）が常に0件になる

期待値の式を展開すると、オッズが約分されて消える:

```
EV = (暗黙勝率 × 倍率 / 100) × odds
   = (0.75 / odds × 倍率) × odds
   = 0.75 × 倍率           ← odds が消える
```

- 11番人気以上でも `EV = 0.75 × 1.40 = 1.05`
- ◆の条件「EV ≥ 1.05」を**数学的にほぼ満たせない**
- 結果として、Proでログインしても「高期待値穴馬」セクションが空になる

### 5.2 期待値がオッズに依存しない

EV が「人気補正倍率」だけで決まるため、本来オッズが高い（妙味のある）馬ほど高EVになるべきところが平坦化している。妙味の比較が機能していない。

## 6. 今後の改善方針（候補）

| 案 | 内容 | 影響範囲 |
|---|---|---|
| A | ◆の閾値・倍率を調整し穴馬が出るようにする | データ側スクリプトのみ |
| B | 現状維持（実データの実態として穴馬なしと表示） | なし |
| C | 学習済みAIモデル（`inference/run_inference.py`）で星評価を付与する運用へ切替 | データ側パイプラインのみ |

いずれの案もフロント（`source/`）は不変で、データ側スクリプト／パイプラインのみで対応可能。
